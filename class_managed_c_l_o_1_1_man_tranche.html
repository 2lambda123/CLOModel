Detailed Description

Class used to describe a single tranche of a deal.

Backward Compatibility Notes
The Read method can load files written by versions 1.73 and later of the model.
According to the version you load the results will change according to the following list:
  1. 1.74 - Payment frequency will be loaded as a single value vector with no anchor date

double ManagedCLO::ManTranche::GetCoupon ( int  index)
inline

The coupon paid by the tranche.

  • index The time index for which to retrieve the tranche coupon. 0 corresponds to the Coupon vector anchor date or, if it's missing, the last payment date.

If the tranche is fixed rate this is the interest paid on the note, if the tranche is floating rate this is the margin over the base rate.

Note
If the tranche is floating rate, the output of this property will be the sum of the base interest rate and the margin.

void ManagedCLO::ManTranche::GetDataFromBloomberg ( )
inline

Downloads tranche data from Bloomberg.

Downloads from Bloomberg the following informations on the tranche:

  • Original Nominal Amount
  • Current Outstanding Amount
  • Currency
  • Last IPD
  • Type of interest (fixed or floating)
  • Payment Frequency
  • Interest Accrued
  • Settlement Date
  • For floating rate tranches:
    • Base rate ticker
    • Spread paid by the note
  • For fixed rate tranches:
    • Ticker of the base rate used to calculate the discount margin
    • Coupon paid by the note

String^ ManagedCLO::ManTranche::Coupon

Set the coupon or margin paid by the tranche.

If the tranche is fixed rate this is the interest paid on the note, if the tranche is floating rate this is the margin over the base rate.
Accepts Bloomberg vectors. If the anchor date for the vector is not specified, the last payment date will be used.
The format is in basis points 5%=500.

int ManagedCLO::ManTranche::ProrataGroup
getset

The seniority group of the tranche.

This determines the seniority of the tranche in the waterfall, tranches that share the same ProrataGroup are considered to be pro-rata.
Values less or equal to 0 are invalid. The higher this value the more junior the tranche is.
The most senior tranche should have this value set at 1.
