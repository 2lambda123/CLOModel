Detailed Description

Class for managing bi-dimensional stress test for tri-dimensional model.

This class allows to stress two of the three assumption on the pool of loans: CPR, CDR and loss severity.
The parameters being stressed lay on two dimensions, called X and Y while the third parameter remains constant

Note
If the waterfall structure has ManWaterfall::UseCall set to true the stress will consider the call scenario.

void ManagedCLO::ManStressTest::SaveResults ( String^  DestPath)
inline

Saves the stress results.

  • DestPath the path to the folder where the results will be saved

This function saves the results of the stress in fcsr format for fast access.
The fcsr format is a compressed file containing the different scenarios and a file containing the general information of the structure, this allows fast access to single scenarios of the stress test without having to load all of them in the memory.

Warning
Calling two instances of this function at the same time with the same DestPath will cause undefined behavior

bool ManagedCLO::ManStressTest::UseMultithread
getset

Determines if multi-threading calculation should be used.

This property determines if the model can leverage multiple CPU cores to speed up calculations.
The number of threads will automatically adapt to the number of cores of the machine (both real and logical)
For single core machines or if the number of cores can't be evaluated (e.g. some virtual machines) this property has no effect and all calculations will be sequential.
By default it's set to true.

Note
If this property is set to true your CPU usage will reach 100% during the computation.
